\chapter{Determine Scope}
\textbf{Purpose:} The ontology models a modern streaming platform ecosystem. It integrates production data (creators, release dates) with a robust content catalog (Films, Series, Episodes, Age Ratings) and consumer access data (streaming platforms, subscriptions, viewing history) to support intelligent content recommendation and access querying.

\vspace{0.5cm}
\noindent\textbf{Competency Questions:}
\begin{enumerate}
    \item Which films and series are currently available on a specific streaming platform (e.g., Netflix)?
    \item Which subscribers meet the age requirements to view specific ``TV-MA'' or ``R'' rated content?
    \item How many episodes belonging to a specific series has a user watched?
    \item What is the average rating of content within the ``Science Fiction'' genre?
    \item Which subscribers are paying for a Premium subscription plan but haven't watched any content this month?
\end{enumerate}


\chapter{Consider Reuse}
For this project, I have chosen to extend an existing ontology rather than building one from scratch.

\begin{itemize}
    \item \textbf{The ontology used:} The Film Ontology.
    \item \textbf{Source/Link:} \url{http://semantics.id/ns/example/film/}
    \item \textbf{Author(s):} Fajar J. Ekaputra (Contributors: Filip Kovacevic, Laura Waltersdorfer, Marta Sabou).
    \item \textbf{Extension Strategy:} The base ontology effectively models film creation (artists, studios). I will extend it by adding a ``Consumer/Streaming'' domain to model where these films and series are hosted, the episodic nature of content, robust content classification, and user viewing habits.
\end{itemize}

    \item \textbf{Descriptive nouns (candidate data properties):}
    \begin{itemize}
        \item \textit{From Base:} releaseYear, fullName, dateOfBirth, budget.

    \end{itemize}
